Step 1 — Run the Database Migration

1

Open MySQL command line (or MySQL Workbench)

mysql -u root -p college_portal < ai_attendance_migration.sql

Or paste the contents of ai_attendance_migration.sql directly into MySQL Workbench and run it.

This creates: ai_attendance_sessions table descriptor column in photos table

📦

Step 2 — Install Dependencies

1

Open terminal in your project folder

cd portal_fixed
2

Install Jest (for unit tests only — optional)

npm install

This installs everything in package.json including jest. The server itself needs no new npm packages — face-api.js loads from CDN in the browser.

🚀

Step 3 — Start the Server

1

Start normally (same as before)

npm start
# Output:
# ✅ MySQL connected
# 🚀 Portal → http://localhost:3000

No changes to startup. All existing pages work identically.

🤖

Step 4 — Use AI Attendance

1

Login as Admin

Open http://localhost:3000 → Login with Admin credentials (e.g. ADMIN001 / admin123)

2

Go to AI Attendance

In the Admin sidebar, click "🤖 AI Attendance" — or open http://localhost:3000/ai_attendance.html

3

Select Branch + Semester

The system auto-detects today's active timetable slot. If no class is running right now, pick a slot manually from the dropdown.

4

Click "Start AI Session" → Load Models

Click "⬇ Load Models & Student Data". face-api.js (~6MB) loads from CDN. Requires internet on first load, cached after. Then student face descriptors load from your DB.

⚠️ Students need at least 3–5 photos registered (via existing registration flow) for reliable recognition. Descriptors are auto-computed when photos are uploaded.
5

Start Camera → Capture Start → Capture End

Allow camera access in browser. Point at classroom. Click "📸 Capture Start" at class beginning, "📸 Capture End" at class end.

6

Finalize & Save

Review the present/absent list. Click "💾 Finalize & Save to DB". Attendance is written to the attendance table exactly as the existing faculty manual process does — students and faculty dashboards update automatically.

Troubleshooting

!

"No students with face descriptors found"

This happens because existing photos don't have descriptors yet. The descriptor is computed in the browser when a new photo is uploaded. For existing students, re-upload one photo via the student registration page — the system will compute and store the descriptor automatically.

!

"No timetable slots found for today"

Upload a timetable CSV first via Admin → Upload Timetable. Make sure the timetable has the correct day_of_week entries (Monday–Saturday) and the batch is marked Active.

!

Models fail to load (network error)

face-api.js model weights load from cdn.jsdelivr.net. Requires internet access. For offline use, download models to public/models/ and change MODEL_URL in ai_attendance.html to '/models/'. Download from: https://github.com/vladmandic/face-api/tree/master/model

!

Camera not working

Browser requires HTTPS or localhost for camera access. Since you're running on localhost:3000 this works. If deploying to a server, add HTTPS (use nginx + certbot).

!

Low recognition accuracy

  • Register 5–7 photos per student from different angles (front, left, right, slight up/down)
  • Ensure good lighting in classroom
  • Lower the threshold in ai_attendance.html: change 0.55 to 0.6 (more lenient) or 0.5 (stricter)
  • For dark classrooms, increase brightness or add lighting
